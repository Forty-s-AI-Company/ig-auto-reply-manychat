# Shadcn Skill

- Use this when implementing dialogs, dropdowns, toasts, sheets, tabs, or form
  controls in the existing UI.
- Prefer consistent composition patterns over one-off custom popovers.
- Use existing local components first; add new wrappers only if reuse is real.
- Disabled controls must still explain why they are unavailable.

## InboxPilot-specific rules

- Dialogs should be small, operational, and direct.
- Toasts should report success, failure, or next action in plain product terms.
- Menus and filters should preserve current workspace or Instagram scope.
